# Contributing to Tech Reborn Wiki

Thank you for your interest in contributing to the Tech Reborn Wiki! This repository uses Docusaurus to generate static documentation.

## Guidelines

1. **Never Invent Recipes**: Always use actual generated data from `TechReborn` source code. If a recipe is missing from the data, leave a `<!-- VERIFY -->` comment and do not hardcode it unless it's a confirmed vanilla integration.
2. **Use Shared Components**: For any item icons, use `<ItemIcon id="techreborn:item_id" />`. For recipes, use `<RecipeFromData id="techreborn:crafting/item_id" />`. For machines, use `<MachineStats>` and `<MachineRecipeList>`.
3. **Keep Tone Neutral**: Maintain a technical and objective tone. Do not add subjective tips unless specifically creating a "Guides" page.

## Adding a New Page

1. Locate the correct folder under `docs/` for your topic (e.g. `docs/multiblocks/`).
2. Follow the template found in the corresponding `plans/done/` file if applicable.
3. Ensure the page has correct Frontmatter:
   ```yaml
   ---
   title: Page Title
   description: Short description.
   sidebar_position: 1
   ---
   ```
4. If you add a new directory, update `sidebars.ts` to use `{ type: "autogenerated", dirName: "your-dir" }`.

## Updating Game Data

We extract data directly from the mod's source code to ensure accuracy.
To update data (e.g. for a new MC version):
1. Clone the Tech Reborn mod repository locally.
2. Ensure you have run `./gradlew runDatagen` in the mod repo.
3. Run `node scripts/extract-tr-data.mjs --tr-source /path/to/TechReborn` to copy textures and compile `src/data/recipes.json` and `src/data/items.json`.
4. Run `npm run docusaurus docs:version NEW_VERSION` if starting a new Minecraft version.

## Pull Requests

1. Fork the repository and create your branch from `main`.
2. Make your edits and ensure the build succeeds locally: `npm run build`.
3. Fix any broken links reported by Docusaurus.
4. Push to your fork and submit a Pull Request.

Happy documenting!
